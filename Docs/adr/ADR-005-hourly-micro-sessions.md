# ADR-005: hourly micro-sessions вместо container restarts

Status: Accepted

## Контекст

Отчеты и калибровка требуют чистого разделения по сессиям и часам. Утренняя, основная, вечерняя и выходная сессии не должны смешиваться. При этом физический hourly restart `trade-core` ухудшит качество потоков и создаст искусственные разрывы.

## Решение

Каждый час закрывается как логическая `micro-session` внутри текущей биржевой сессии. `trade-core` остается живым. Micro-session закрывается на часовую границу или на границу биржевой сессии.

## Последствия

- Если робот стартовал в середине часа, первая micro-session будет короткой.
- Если биржевая сессия заканчивается раньше полного часа, micro-session закрывается на границе сессии.
- `session_run_closed` запускает report pipeline.
- События должны хранить `session_type`, `session_phase`, `micro_session_id`, `calendar_date`, `trading_date`, `broker_trading_status`.
