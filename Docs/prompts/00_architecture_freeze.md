# Step 00 Prompt: Architecture Freeze

Ты работаешь в репозитории торгового робота для торговли на Московской бирже через T-Invest API.

Обязательные инварианты проекта:

1. Бэкенд полностью на Python.
2. Фронтенд на Vue 3 в dark theme.
3. Основной критический путь — долгоживущий контейнер trade-core; не делай физический hourly restart trade-core.
4. T-Bank gRPC — основной брокерский транспорт; WebSocket используй для стриминга и фронтенд-BFF, но не подменяй им все торговые методы.
5. Postgres — источник истины по состоянию, ордерам, событиям стратегии, отчётам и аудиту.
6. Redis — брокер фоновых задач и лёгкий coordination/cache слой.
7. Prometheus + Grafana — метрики и дашборды; Loki + Fluent Bit — технические логи.
8. Технические сырые логи не храни как основной источник аналитики в файлах или в Postgres; доменные события и агрегаты храни в Postgres, технические логи отправляй в Loki.
9. Часовые сессии — логические micro-sessions внутри биржевой торговой сессии.
10. Любые тяжёлые отчёты и дневная аналитика должны выполняться через report-worker, а не в процессе API.
11. Система логирования должна быть пригодна для последующей машинной аналитики и counterfactual-разбора заблокированных/отменённых сделок.
12. Никогда не коммить секреты. Для production используй Docker Compose secrets. Для dev допустим fallback через локальные env, но не через закоммиченные реальные ключи.
13. По каждой задаче обновляй Docs/ и тесты. Не оставляй архитектурные решения только в коде.
14. По окончании каждого шага покажи изменённые файлы, команды запуска, добавленные тесты, риски и ограничения.
